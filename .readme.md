# Sistema de Ponto Eletrônico — API REST

API RESTful desenvolvida para a disciplina de **Desenvolvimento Web 2026**. O sistema permite o gerenciamento de colaboradores e o controle de horários de entrada e saída em tempo real.

A arquitetura segue o padrão em camadas (MVC) para garantir baixo acoplamento, alta coesão e facilidade de manutenção.

---

## Stack de Tecnologias Utilizadas

| Tecnologia | Finalidade |
|---|---|
| Java 21 | Linguagem principal |
| Spring Boot 3.x | Framework base da aplicação |
| Spring Data JPA | Persistência de dados |
| H2 Database | Banco de dados em memória (desenvolvimento) |
| Spring Validation | Validação das requisições |
| SpringDoc OpenAPI (Swagger) | Documentação interativa da API |
| Docker | Conteinerização da aplicação |
| Maven | Gerenciador de dependências |

---

## Estrutura de Pastas

```
src/main/java/com/app/
├── controller/   # Recebe requisições HTTP e retorna respostas JSON
├── service/      # Regras e lógica de negócio
├── repository/   # Persistência e comunicação com o banco via JPA
├── model/        # Entidades que representam as tabelas do banco
├── dto/          # Objetos de transferência de dados com validação
└── infra/        # Tratamento global de exceções e erros HTTP
```

## Executando com Docker (Recomendado)

> Não é necessário ter Java ou Maven instalados na máquina.

**Pré-requisito:** Docker instalado.

**1. Construa a imagem:**
```bash
docker build -t trabalho-web-api .
```

**2. Inicie o container:**
```bash
docker run -p 8080:8080 trabalho-web-api
```

A API estará disponível em: `http://localhost:8080`

---

## Executando Localmente (Sem Docker)

**Pré-requisitos:** Java 21 e uma IDE de sua preferência (VS Code, IntelliJ, Eclipse).

**Execute o servidor embutido Tomcat via Maven:**
```bash
./mvnw spring-boot:run
```

A aplicação estará ativa em: `http://localhost:8080`

## Endpoints da API

Todas as requisições e respostas utilizam o formato **JSON**.

### Colaboradores

#### `POST /api/usuarios` — Cadastrar Colaborador

Cria um novo colaborador no sistema.

**Corpo da requisição:**
```json
{
  "nome": "Mateus Lopes",
  "email": "mateus@email.com"
}
```

**Resposta:** `201 Created`

---

#### `GET /api/usuarios` — Listar Colaboradores

Retorna a lista completa de colaboradores cadastrados.

**Resposta:** `200 OK`

---

### Registro de Ponto

#### `POST /api/pontos/bater` — Bater Ponto

Registra a data e hora exata do servidor para o usuário informado.

**Corpo da requisição:**
```json
{
  "usuarioId": 1
}
```

**Resposta:** `201 Created`
```json
{
  "id": 1,
  "dataHora": "2026-05-26T20:54:00",
  "usuario": {
    "id": 1,
    "nome": "Mateus Lopes",
    "email": "mateus@email.com",
    "ativo": true
  }
}
```

---

#### `GET /api/pontos/usuario/{id}` — Histórico de Pontos

Retorna todas as batidas de ponto de um colaborador, ordenadas da mais recente para a mais antiga.

**Resposta:** `200 OK`
```json
[
  {
    "id": 1,
    "dataHora": "2026-05-26T20:54:00"
  }
]
```

## Tratamento de Erros

A API retorna respostas de erro estruturadas em JSON com o status HTTP correspondente.

**`400 Bad Request` — Erro de validação** (ex: campo `email` inválido ou vazio):
```json
{
  "status": 400,
  "erro": "O campo email é obrigatório",
  "campo": "email"
}
```

**`404 Not Found` — Entidade não encontrada** (ex: `usuarioId` inexistente):
```json
{
  "status": 404,
  "erro": "Usuário não encontrado com o ID fornecido."
}
```
